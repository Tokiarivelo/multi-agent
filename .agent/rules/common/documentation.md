# Documentation Requirements

For any change:
- Update or create `.md` documentation files.
- Explain purpose, flow, events, dependencies.
- Document environment variables (`env`).
- Update `README.md` if the overarching architecture or run commands change.
- Always generate multi-language support (en-us and fr) if documentation is meant for end-users.

**Undocumented features are forbidden.**
